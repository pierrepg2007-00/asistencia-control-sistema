# Guía del proyecto para agentes

## Objetivo del proyecto

`control_academico_web` es una web sencilla para apoyar el control académico de estudiantes, materias, matrículas, notas, asistencia y reportes.

## Tecnologías usadas

- Python para la lógica principal.
- HTML para la estructura de las páginas.
- CSS para estilos básicos.
- JavaScript básico para interacción en la web.
- JSON para guardar los datos.

## Estructura de carpetas

- `control_academico_web/data/`: archivos JSON del sistema.
- `control_academico_web/core/`: lógica en Python.
- `control_academico_web/web/`: páginas HTML.
- `control_academico_web/static/`: archivos CSS y JavaScript.
- `control_academico_web/evidencias/`: evidencias, bitácoras y pruebas.
- `control_academico_web/reportes_generados/`: reportes exportados o generados.
- `control_academico_web/prompts/`: prompts usados por cada integrante.

## Reglas de desarrollo

- Trabajar solo en el módulo asignado.
- No borrar archivos de otros integrantes.
- Mantener el código simple y entendible.
- Registrar los prompts usados y las evidencias del resultado.
- No subir tokens, contraseñas ni archivos sensibles.
- Guardar los datos en archivos JSON cuando corresponda.

## Módulos del sistema

1. Estudiantes.
2. Materias y periodos.
3. Matrículas.
4. Notas.
5. Asistencia.
6. Reportes.

## Cómo ejecutar el proyecto

Por ahora el proyecto está en etapa inicial. Los archivos HTML pueden abrirse desde la carpeta `web/` y la lógica Python puede probarse desde la carpeta `control_academico_web/`.

## Definición de terminado

Un avance se considera terminado cuando:

- El archivo solicitado fue creado o modificado.
- El código funciona de forma básica.
- No se desarrollaron módulos fuera del alcance.
- El prompt usado quedó registrado.
- La evidencia del resultado quedó actualizada.
- Los cambios fueron revisados antes del commit.
